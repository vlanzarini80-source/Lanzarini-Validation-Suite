## Purpose

This validation verifies the structural integrity of the public repository.

The objective is to automatically detect missing files, missing directories, empty documentation files, and Python syntax errors before future repository updates.

---

## Configuration

The workflow automatically checks:

- required public documentation files;
- required repository directories;
- non-empty Markdown documentation;
- Python syntax validation using `compileall`.

- required files exist;
