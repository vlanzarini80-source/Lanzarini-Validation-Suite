# V43A: End-to-End vLLM Adapter Performance Analysis

## Scope

This document reports the preserved experimental results of the V43A end-to-end adapter performance analysis.

Only experimentally observed results are reported.

No estimated, reconstructed or fabricated values are included.

Whenever the original JSON or CSV artifacts are not currently available, only preserved numerical summaries are reported.

---

## Purpose

The objective of this checkpoint was to measure the end-to-end throughput of the adapter-based integration and to identify the dominant performance bottleneck.

This checkpoint evaluated the complete inference path rather than the Triton kernel in isolation.

---

## Hardware

- GPU: NVIDIA A100-SXM4-80GB

---

## Preserved Performance Results

| Metric | Value |
|--------|------:|
| FlashAttention baseline throughput | 69.04 tokens/s |
| Adapter replacement throughput | 9.03 tokens/s |
| reconstruct_cache() time | 2.293 ms per q=1 step |

---

## Interpretation

The preserved measurements indicate that:

- the adapter replacement path achieved approximately 9.03 tokens/s;
- the FlashAttention baseline achieved approximately 69.04 tokens/s;
- the dominant overhead originated from paged-to-contiguous cache reconstruction.

The preserved project analysis concluded that the Triton kernel itself was not the primary performance bottleneck in this experimental configuration.

---

## Limitations

These measurements apply only to the evaluated implementation.

They do not establish:

- end-to-end performance for future paged-native implementations;
- performance on different hardware;
- performance for different transformer architectures;
- universal throughput characteristics.
