# V58B — Measured Backend-Winner Extraction

## Scope

This document reports the preserved outcome of the V58B backend-winner extraction checkpoint.

Only experimentally observed winner information preserved from the original benchmark campaign is included.

No missing latency values, grid dimensions, or row-level measurements are reconstructed.

---

## Purpose

V58B analyzed the measured V58A runtime results to identify which backend was faster for each evaluated sparse-local attention configuration.

The compared backends were:

- Lanzarini sparse-local attention;
- FlashAttention sliding-window attention.

This checkpoint extracted winners from measured runtime data. It did not introduce a new performance experiment.

---

## Hardware and Experimental Context

The preserved project record identifies the experimental context as:

- GPU: NVIDIA A100-SXM4-80GB
- Precision: FP16
- Head dimension: 64
- Measurement type: forward runtime
- Comparison: Lanzarini versus FlashAttention sliding-window

---

## Preserved Winner Pattern

The preserved result records FlashAttention as the measured winner only at the following configurations:

| Sequence length T | Window W | Measured winner |
|---:|---:|---|
| 12288 | 512 | FlashAttention |
| 24576 | 512 | FlashAttention |

For the remaining configurations in the evaluated V58A measurement grid, the preserved result records:

- measured winner: Lanzarini.

---

## Interpretation

The extracted winner pattern shows that the fastest backend depended on the evaluated configuration.

Within the measured V58A grid:

- FlashAttention won at the two preserved `W = 512` configurations listed above;
- Lanzarini won the remaining tested configurations.

This is evidence of regime-dependent backend performance on the evaluated grid.

It is not evidence that `W = 512` universally implies a FlashAttention win.

---

## Limitations

The currently preserved summary does not provide:

- the complete row-level V58A table;
- exact latency values for every configuration;
- the complete list of tested sequence lengths and windows;
- uncertainty estimates for each winner;
- original byte-identical JSON or CSV artifacts.

Those details are therefore intentionally omitted.

The observed winner pattern must not be generalized to:

- untested sequence lengths;
- untested window sizes;
- different GPU architectures;
- different software versions;
- different batch sizes;
- different head dimensions;
- different numerical precisions.

V58B preserves the following limited experimental conclusion:

> In the evaluated V58A grid, FlashAttention was the measured winner at `(T = 12288, W = 512)` and `(T = 24576, W = 512)`, while Lanzarini was recorded as the winner for the remaining tested configurations.

No broader claim is supported by the currently preserved summary.
